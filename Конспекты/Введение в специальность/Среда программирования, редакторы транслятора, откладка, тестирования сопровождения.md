### Терминал

Представляет из себя программу за частую (системную) для выполнения текстовых команд. Для выполнения текстовых команд. Библиотека и framework. Библиотека представляет собой набор готов функций, для решения определённых задач. Framework это каркас или шаблон предоставляющий готовый шаблон инструментов, библиотек и правил для установки приложений. ОБ платформа например GitHub.

### Системы автоматической сборки

- **Gradle**. Позволяет управлять процессом сборки и зависимостями проектов. Поддерживает множество языков программирования, включая Java, Kotlin, Groovy и другие. Обладает механизмом кэширования, который ускоряет сборку проекта.
- **Apache Maven**. Одна из самых распространённых систем сборки. Использует XML для описания процесса сборки и зависимостей. Поддерживает управление жизненным циклом проекта.
- **Apache Ant**. Инструмент для автоматизации процесса сборки, написанный на Java. Использует XML для определения задач и зависимостей.
- **Bazel**. Разработан Google и предназначен для построения больших, сложных проектов. Поддерживает различные языки программирования и ориентирован на повторяемость сборочных процессов.
- **CMake**. Часто используется в проектах на C и C++. Позволяет определять процесс сборки с использованием файлов CMakeLists.txt.
- **SBT (Simple Build Tool)**. Часто применяется в проектах, написанных на Scala. Использует файлы сценариев на Scala для описания сборочных задач и зависимостей.
