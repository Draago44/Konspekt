#  Приведение типов в Python (Type Conversion / Typecasting)

> [!summary] Определение
> Процесс изменения типа данных значения для соответствия требованиям операции или хранения. Python — **динамически**, но **строго типизированный** язык: автоматическое смешивание несовместимых типов запрещено.

---

##  1. Неявное приведение (Implicit)
Выполняется **интерпретатором автоматически** при операциях с совместимыми типами.

```python
a = 10      # int
b = 3.14    # float
c = a + b   # int → float автоматически
print(type(c))  # <class 'float'>
```

**Характеристики:**
-  **Безопасность:** Всегда безопасно, данные не теряются
-  **Ограничения:** Только для совместимых типов (`int` ↔ `float`, `bool` → `int`)
-  **Контроль:** Отсутствует, управляется интерпретатором

> [!tip] Правило
> Python всегда приводит значение к более «широкому» типу, чтобы избежать потери точности.

---

##  2. Явное приведение (Explicit)
Выполняется **вручную** через встроенные функции-конструкторы. Даёт полный контроль, но требует валидации.

**Основные функции:**
- `int()` — в целое число. Пример: `int("42") → 42`, `int(3.9) → 3`
- `float()` — в число с плавающей точкой. Пример: `float("3.14") → 3.14`, `float(5) → 5.0`
- `str()` — в строку. Пример: `str(100) → "100"`
- `bool()` — в логическое значение. Пример: `bool(0) → False`, `bool("hi") → True`
- `list()` / `tuple()` / `set()` — в коллекции. Пример: `list("abc") → ['a', 'b', 'c']`
- `dict()` — в словарь. Пример: `dict([(1,"a")]) → {1: 'a'}`

```python
age = input("Возраст: ")  # input() всегда возвращает str
age_int = int(age)        # явное приведение
```

##  Практические рекомендации
- `int()` **отбрасывает** дробную часть, не округляет. Для округления используй `round()`, `math.floor()`, `math.ceil()`.
- `bool()` удобно для фильтрации: `list(filter(None, data))` удалит все falsy-значения (`0`, `""`, `[]`, `{}`, `None`).
- При `dict()` из списка каждый элемент должен быть итерируемым объектом **ровно из 2 элементов** `(key, value)`.
- Ввод из `input()`, чтение из файлов/CSV/API почти всегда возвращает `str` → требуй явного приведения.

---

##  Сравнение неявного и явного
- **Инициатор:** Неявное → интерпретатор | Явное → программист
- **Контроль:** Неявное → отсутствует | Явное → полный
- **Безопасность:** Неявное → всегда безопасно | Явное → возможны `ValueError` / `TypeError`
- **Потеря данных:** Неявное → нет | Явное → возможна (например, `float → int` отбрасывает дробную часть, `dict()` перезаписывает дубликаты ключей)
- **Гибкость:** Неявное → ограничена совместимыми типами | Явное → высокая
