#SSH
##  Что это
SSH — это протокол для безопасного удалённого доступа к серверу или компьютеру через незащищённую сеть.

Используется для:
- удалённого управления сервером
- передачи файлов
- туннелирования соединений

***

##  Как работает
SSH использует:
- шифрование (обычно через RSA, Ed25519)
- аутентификацию (пароль или ключи)

Схема:
1. Клиент подключается к серверу
2. Проверяется подлинность сервера
3. Устанавливается защищённое соединение
4. Происходит вход (пароль или ключ)

***

##  Аутентификация

### 1. По паролю
Просто ввод пароля пользователя:
```
ssh user@host
```

Минус: менее безопасно

***

### 2. По ключам (рекомендуется)

Генерация ключа:
```
ssh-keygen -t ed25519
```

Копирование на сервер:
```
ssh-copy-id user@host
```

После этого вход:
```
ssh user@host
```

Без пароля 

***

##  Основные файлы

- `~/.ssh/id_ed25519` — приватный ключ
- `~/.ssh/id_ed25519.pub` — публичный ключ
- `~/.ssh/authorized_keys` — ключи, которым разрешён доступ
- `~/.ssh/config` — конфигурация SSH

***

##  Конфиг (упрощение подключения)

Файл: `~/.ssh/config`

Пример:
```
Host myserver
    HostName 192.168.1.10
    User draago
    Port 22
    IdentityFile ~/.ssh/id_ed25519
```

Теперь можно подключаться:
```
ssh myserver
```

***

##  Передача файлов

### SCP
```
scp file.txt user@host:/path/
```

### Rsync (быстрее и удобнее)
```
rsync -avz file.txt user@host:/path/
```

***

##  Полезные команды

- Проверка подключения:
```
ssh -v user@host
```

- Указать порт:
```
ssh -p 2222 user@host
```

- Выполнить команду удалённо:
```
ssh user@host "ls -la"
```

***

##  Безопасность

- Использовать ключи вместо паролей
- Отключить root-доступ
- Сменить стандартный порт (22)
- Использовать fail2ban

***

##  Где используется

- DevOps и администрирование
- Работа с VPS / облаками
- Git (через SSH)
- Автоматизация и CI/CD

***
